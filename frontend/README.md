## вҡЎ Design Decisions

Every technical decision was intentional:

| Decision | Reason |
|---|---|
| **No chart libraries (Recharts / D3)** | Custom SVG gives full control + zero bundle bloat |
| **Single-page architecture** | Faster interactions, no routing overhead |
| **Markdown вҶ’ UI parser (custom)** | LLM output is raw text вҖ” structured display requires deterministic parsing |
| **JetBrains Mono for data labels** | Monospace enforces visual alignment in tables and metrics |
| **CSS-only animations** | No Framer Motion dependency вҖ” keeps the bundle lean |
| **Keyboard-first input design** | Power users don't want to reach for the mouse |
| **Session history in state** | No backend needed for recent audits вҖ” instant UX |

> Built for speed, clarity, and developer control вҖ” not just feature count.

## рҹ§  Key Learnings

Building GrowthLens taught me things no tutorial covers:

- **Designing hybrid AI systems** вҖ” combining deterministic SEO crawling with probabilistic LLM output, and making both trustworthy together
- **Parsing LLM responses into UI** вҖ” raw markdown is useless; you need a structured parser that maps AI output to typed components
- **Performance-first frontend** вҖ” building radar charts, sparklines, and animated rings in raw SVG instead of reaching for a library by default
- **Keyboard-first UX design** вҖ” building `вҢҳK` / `вҢҳвҶө` shortcuts from scratch and understanding why power users need them
- **Prompt engineering for structured data** вҖ” making the LLM return consistent, parseable section headers across every audit

> These are the exact things interviewers ask about in system design and frontend architecture rounds.
